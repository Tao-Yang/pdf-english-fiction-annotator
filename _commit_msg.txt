fix(selector): make CEFR dictionary tiers a true nested hierarchy

- Rank non-priority picks by rarity (Zipf frequency) before applying the
  per-page density cap, so when a page has more qualifying candidates than
  the cap, the rarest words always win regardless of tier. This removes the
  arbitrary encounter-order truncation that let A2/B1/B2 pick different,
  unrelated filler words instead of overlapping sets.
- Add a CEFR-scaled notes-per-page cap (CEFR_MAX_NOTES / notes_cap()) so
  easier tiers allow strictly more notes than stricter ones (A2=16, B1=14,
  B2=12, C1=9), matching the requested "由多到少" behavior. B2 keeps its
  previously validated default of 12. max_notes_per_page now defaults to
  None and only overrides notes_cap() when explicitly set via CLI flag.
- Verified via synthetic and real-sample smoke tests that C1 subset B2
  subset B1 subset A2 holds exactly, with A2 always showing the most notes.
