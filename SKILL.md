# 英文小说 PDF 词汇注释

## 目标

生成一份可阅读、可检索的注释版 PDF：

- 保留原始页面、排版、图片、书签和可复制文本；
- 对值得学习的单词和词组给出符合上下文的简明释义；
- 注释放在目标词语右侧的可用空白区域；空间不足时扩展页面右边界，绝不覆盖正文；
- 用细引导线或编号建立词语与注释的明确对应关系；
- 输出独立文件，不覆盖源 PDF。

## 输入与默认值

执行前从用户消息和文件中确定：

1. 输入 PDF 路径；
2. 页码范围，默认全文；
3. 读者英语水平，默认 CEFR B2；
4. 注释语言，默认简体中文；
5. 注释密度，默认“适中”（每个正文页约 5–12 条，按实际文本调整）；
6. 输出路径，默认在源文件同目录生成 `<原文件名>-annotated.pdf`。

若缺少非关键参数，使用默认值继续，不要阻塞任务。只有输入文件、密码或目标词表等关键条件确实缺失时才询问。

## 工具与实现原则

优先使用 Python 与 PyMuPDF（`fitz`），因为它能同时读取词语坐标、扩展页面和绘制文本。缺少依赖时安装 `pymupdf`。必要时使用 OCR，但不得把低置信度 OCR 结果直接当作可靠正文。

所有临时文件放入独立临时目录；任务成功后保留最终 PDF 和可选的注释清单，清理无用中间文件。

## 工作流程

### 1. 检查源 PDF

1. 验证文件存在、可打开、页数合理且未被未知密码锁定。
2. 抽查首页、一个中间页和末页，确认页面方向、裁切框、单双栏、正文区域及右侧空白宽度。
3. 提取带坐标文本：使用 `page.get_text("words")` 和 `page.get_text("dict")`。
4. 若正文页几乎无可提取文本，判定为扫描版并进入 OCR 分支。
5. 记录源文件页数、尺寸、元数据和书签，以便输出后核对。

### 2. 识别小说正文

- 排除页眉、页脚、页码、目录、版权页、章节标题、脚注编号和重复装饰文字。
- 按坐标恢复自然阅读顺序，处理断行连字符，例如将行末 `extra-` 与下一行 `ordinary` 合并为 `extraordinary`。
- 尊重多栏布局；不要跨栏拼接句子。
- 为每个候选词或词组保留：页码、原文、所在句、边界框和行号。

### 3. 选择注释项

结合上下文选择，而不是机械标注所有长词。

优先标注：

- 超出目标读者水平且影响理解的实词；
- 习语、短语动词、固定搭配、文学性表达和旧式用法；
- 在当前语境中含义不同于常见义的熟词；
- 文化负载词、方言、俚语或关键专有概念；
- 对情节或人物刻画有重要作用的表达。

通常不标注：

- `the`、`and` 等基础功能词；
- 可由紧邻上下文轻易推断且不影响理解的词；
- 人名、普通地名和已在附近解释过的重复项；
- 仅因拼写较长、但对目标水平明显简单的词。

去重策略：同一词义默认首次出现时详注；后续仅在含义改变、间隔很远或用户选择“密集”模式时再注。

### 4. 编写释义

每条注释应简短并严格匹配上下文：

- 单词：仅给出简明的中文语境义，必要时补充语气或用法；
- 词组：整体释义优先，不逐词硬译；
- 一词多义：只列当前句所需含义；
- 必要时添加极短提示，如“讽刺”“旧式”“口语”；
- 默认不添加词性、音标、英文释义、例句、长篇词源、整句翻译或与当前语境无关的义项。

建议格式：`① recoil  畏缩；退避`。

### 5. 规划右侧注释位置

对每一页先计算正文和现有内容的联合边界框，再进行布局。

1. 找到目标词所在行的右端 `line_right`。
2. 首选从 `max(target_rect.x1 + gap, line_right + gap)` 开始的同高右侧空白。
3. 将该区域与页面中的文字、图片、矢量图和已有注释框做碰撞检测。
4. 若可用宽度和高度足够，将注释放在该空白区，基线尽量与目标词对齐。
5. 若同一高度存在多个注释，按从上到下顺序排布并保持最小间距；发生冲突时向下寻找最近空位。
6. 若原页面右侧空间不足，扩展页面右边界，创建统一的右侧注释栏。不得缩小、重排、裁切或覆盖原始页面内容。
7. 扩展栏宽度应根据最长注释计算，并设置合理上限；注释过长时换行，不应使用过小字号强行塞入。
8. 用浅灰细引导线连接目标词右缘与注释；线条应避开正文，无法避开时使用同页编号对应，不穿过字形。

禁止：在正文上加不透明文本框、白色遮罩、覆盖式高亮或任何会降低原文可读性的元素。

### 6. 写入 PDF

- 在源 PDF 的副本上处理，使用页面扩展或新页面承载新增右侧区域。
- 优先使用支持中文的嵌入字体；依次查找系统中的 `Microsoft YaHei`、`SimSun`、`Noto Sans CJK SC`。若均不可用，停止并说明缺少中文字体，不要生成乱码文件。
- 中文注释建议 7–9 pt，最低不得小于 6.5 pt；原词可加粗，释义使用常规字重。
- 注释颜色使用深灰或深蓝，确保打印时仍清晰；引导线使用更浅颜色。
- 可选地给目标词添加非常浅的透明下划线或细框，但不得遮挡字形。
- 保留元数据和目录书签；输出文件名不得覆盖源文件。

### 7. 质量检查

完成后必须进行自动检查和视觉抽查。

自动检查：

- 输出可重新打开，页数与源文件一致；
- 每页原始内容区域未被裁切，原始文字数量没有异常下降；
- 每个注释框都位于页面内；
- 注释框与原始文字、图片边界无相交；
- 注释之间无重叠；
- 中文字体已嵌入或能够稳定显示；
- 输出文件不是空文件，且没有覆盖输入文件。

视觉抽查：

- 渲染至少首页正文页、注释最多的一页、一个中间页和末页；
- 确认注释位于对应词语右侧、引导关系明确、无乱码、无截断、无正文遮挡；
- 若任一抽查页失败，修复布局后重新生成并再次检查。

### 8. 交付

向用户报告：

- 输出 PDF 路径；
- 处理页码范围和注释总数；
- 使用的读者水平、注释语言和密度；
- 是否发生 OCR、页面扩展或跳过无法可靠识别的页面；
- 视觉抽查结果及任何已知限制。

除非用户要求，不要在聊天中粘贴整本小说正文或大段受版权保护的原文。

## 分支处理

### 扫描版 PDF

1. 逐页渲染为 300 DPI 左右图像。
2. OCR 时请求词级边界框和置信度。
3. 低置信度候选不自动注释；优先抽查专有名词、连字符和弯引号。
4. 保留原扫描图作为页面底图，只在右侧空白或扩展栏中添加注释。
5. OCR 仅用于定位和理解，不用 OCR 文本覆盖原图。

### 页面没有右侧空白

扩展右侧媒体框或创建更宽的新页面，将原页无缩放地放在左侧，再在新增区域中写注释。所有页面可统一扩展宽度以保持翻页体验一致。

### 注释过多

依次采用：缩短释义、合并同一行的相关词组、适度减小字号、增加右侧栏宽度。仍无法容纳时减少低优先级注释；禁止覆盖原文。

### 加密或损坏

若需要密码，要求用户直接提供可解密版本或在本地终端输入密码。若源文件损坏，停止写入并报告具体页码或解析错误。

## 完成标准

仅当以下条件全部满足时才算完成：

- 释义与小说语境一致；
- 目标词与注释一一可辨；
- 注释在词语右侧的空白处或新增右侧栏；
- 没有任何注释、引导线或背景遮挡原文；
- PDF 可打开、可翻页、中文正常显示；
- 已完成自动检查和代表性页面的视觉抽查；
- 源文件保持不变。
